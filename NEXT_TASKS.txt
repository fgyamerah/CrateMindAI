ARTIST MERGE (completed 2026-04-12)
----------------------------------------------------------------------
[x] (P1) Fix _classify_merge() normalization bug
    Old code compared fi.primary_artist.lower() directly; "Heavy-K" and "Heavy K"
    produced different lowercase strings → incorrectly flagged as AMBIGUOUS.
    Fix: _normalize_primary_for_compare() applies same transforms as normalize_artist_key()
    (strip periods, hyphen→space) before comparison.
    File: modules/artist_merge.py

[x] (P1) Three-category merge classification
    SAFE_ALIAS          — formatting/punctuation/casing differences only
    SAME_PRIMARY_COLLAB — same primary after normalization; ≥1 folder has collab suffix
    AMBIGUOUS           — primary artists genuinely differ; never auto-merged
    File: modules/artist_merge.py

[x] (P2) _describe_alias_differences() helper
    Generates human-readable reason strings for SAFE_ALIAS groups:
    "capitalization variant", "dotted-initials variant", "hyphen/space variant",
    "trailing period variant", "period variant", etc.
    File: modules/artist_merge.py

[x] (P2) _pick_canonical() all-uppercase penalty
    All-uppercase canonical names (HOSH) penalized; mixed-case preferred (H.O.S.H).
    File: modules/artist_merge.py

[x] (P1) Tests for artist merge (77 tests)

PURPOSE
----------------------------------------------------------------------
Automated DJ library preparation:
Raw audio → clean metadata → BPM/key → playlists → Rekordbox export

Design principles:
- Local-first
- Idempotent
- Rekordbox-compatible output
- MIK-first: Mixed In Key is the authoritative source of BPM, key, and cues
- Backend API: FastAPI wrapper around the CLI (Phase 1 complete)

----------------------------------------------------------------------
CORE LOGIC
----------------------------------------------------------------------
HARMONIC MIXING:
- Camelot system
- Strict mode enforces compatible transitions

BPM / KEY HANDLING (MIK-FIRST):
- Mixed In Key is the authoritative source. The pipeline NEVER overwrites
  existing BPM or key values.
- analyzer._read_existing_analysis() reads TBPM/TKEY (MP3), BPM/INITIALKEY
  (FLAC), tmpo/©key (M4A) before running aubio / keyfinder-cli.
- aubio (→ librosa fallback) only runs when BPM is absent in both DB and tags.
- keyfinder-cli only runs when Camelot key is absent in both DB and tags.

CUE POINTS:
- Cue suggest is DISABLED by default (MIK owns cues).
- --force-cue-suggest enables it for non-MIK workflows only.

REKORDBOX XML:
- XML export is DISABLED by default (MIK writes cues into Rekordbox XML).
- --force-xml enables it with a warning. Use only when MIK is not in use.
- M3U playlist generation is always enabled (safe, does not affect MIK state).

SANITIZER (sanitize_text):
- Step 0: strip leading track-index junk before first pipe
  Pattern: `^[^\w\s]*(?:\d+|[A-Za-z])\s*\|\s*`
  Removes "2 | ", "A | ", "***3 | " etc. at start of string only.
  Mid-string pipes and multi-letter words before pipe are untouched.
- Steps 1–9: URL removal, promo phrases, symbol cleanup, artifact cleanup.

IDEMPOTENCY:
- TXXX:PROCESSED=1
- DB status='ok'

AUDIT QUALITY (modules/audit_quality.py — v1.4.0)
----------------------------------------------------------------------
audit-quality subcommand: classify every audio file by codec/bitrate.

Quality tiers:
  LOSSLESS  FLAC / ALAC / WAV / AIFF (codec-based; bitrate ignored)
  HIGH      lossy >= 256 kbps
  MEDIUM    lossy 192–255 kbps  (lower bound: --min-lossy-kbps, default 192)
  LOW       lossy < 192 kbps
  UNKNOWN   ffprobe failure or unrecognized codec

Default behavior: non-destructive (probe + classify + report; no moves, no tag writes)

Optional actions:
  --move-low-quality DIR   move only LOW files; relative folder structure preserved
  --write-tags             write QUALITY tag per file
    MP3  : TXXX:QUALITY (ID3v2.3)
    FLAC : QUALITY Vorbis comment
    M4A  : ----:com.apple.iTunes:QUALITY MP4 freeform atom
    AIFF/WAV: skipped safely
  --dry-run                log intentions; no file writes; no report files

Report output: logs/reports/audit_quality/audit_quality_TIMESTAMP.{csv,json}
DB: quality_tier TEXT column added to tracks table via migration in init_db()

ONLINE METADATA ENRICHMENT (intelligence/enrichment/ — 2026-04-18)
----------------------------------------------------------------------
metadata-enrich-online subcommand: fill missing album/label/ISRC tags via
Spotify Web API (primary) and Deezer API (fallback). Preview-first; --apply
to write. Never writes artist field.

Package layout (intelligence/enrichment/):
  enrichment_schema.py  — EnrichmentCandidate, EnrichmentMatch dataclasses
  metadata_matcher.py   — weighted similarity scoring + ISRC exact match + change policy
  spotify_lookup.py     — SpotifyClient (client credentials OAuth2, token cache)
  deezer_lookup.py      — DeezerClient (public API, no auth)
  runner.py             — orchestrator: search → score → preview/apply → log

Scoring weights:
  artist_sim  0.35   title_sim (base, version stripped)  0.30
  version_sim 0.25   label_sim  0.10
  ISRC exact match → confidence 0.98 (overrides formula)
  Label alias boost: +0.05 when current + candidate labels are in the same alias group

Min confidence threshold: ENRICH_ONLINE_MIN_CONFIDENCE = 0.80 (was 0.92)
  - Overridable via ENRICH_ONLINE_MIN_CONFIDENCE env var or --min-confidence flag

Hard safeguards (metadata_matcher.best_match — applied after scoring):
  1. Version mismatch: if BOTH titles carry explicit but DIFFERENT version tokens:
       confidence capped at 0.74, proposed_changes cleared, reason prefixed
       "VERSION MISMATCH BLOCKED".  Missing version on either side → NOT a mismatch.
  2. Low artist similarity: if artist_sim < 0.90 (and no ISRC anchor):
       confidence capped at 0.74.  Proposed changes KEPT for preview.
       Reason prefixed "LOW ARTIST SIM (x.xx) — auto-apply blocked".
  Neither safeguard fires when an ISRC exact match (0.98) anchors the result.

Change policy (safety rules):
  artist       : NEVER proposed
  title        : only if current is blank, or ISRC matched + base differs
  album        : if current is empty, or ISRC matched
  label        : if current is empty; or ISRC matched + conf >= 0.95
  isrc         : only if not already set

Dataset logging:
  AI_ENRICH_QUEUE    = data/intelligence/enrichment_queue.jsonl    (every result)
  AI_ENRICH_ACCEPTED = data/intelligence/enrichment_accepted.jsonl (applied only)
  AI_ENRICH_REJECTED = data/intelligence/enrichment_rejected.jsonl (skipped/errors)

Source priority (runner._search_with_fallback):
  1. Spotify ISRC       — most precise; skipped when file has no ISRC
  2. Spotify artist+title
  3. Deezer             — fallback when Spotify conf < 0.70
  4. Traxsource         — dance-music specialist fallback; triggered when:
                          a) best-so-far < 0.80, OR
                          b) genre tag contains house/afro/deep/soulful/amapiano/etc.
  Winning = highest confidence across all sources tried.
  EnrichmentMatch.sources_tried records which APIs were queried.

Extension points (not yet implemented):
  - Album-detail API call for Spotify/Deezer to retrieve record label
  - Traxsource CSS selectors should be verified against live site before production

Config keys:
  SPOTIFY_CLIENT_ID, SPOTIFY_CLIENT_SECRET (env var backed)
  ENRICH_ONLINE_MIN_CONFIDENCE = 0.80
  AI_ENRICH_{QUEUE,ACCEPTED,REJECTED}

ARTIST MERGE (modules/artist_merge.py — v1.7.0)
----------------------------------------------------------------------
artist-merge subcommand: consolidate artist folder spelling variants.

Three-category classification:
  SAFE_ALIAS          Difference is formatting/punctuation/casing only.
                      Examples: "Heavy-K"/"Heavy K", "H.O.S.H"/"HOSH",
                      "Rosalie"/"Rosalie.", "culoe de song"/"Culoe De Song"
  SAME_PRIMARY_COLLAB Primary artist matches after normalization; ≥1 folder
                      has a feat/ft/comma collab suffix.
                      Examples: "Heavy-K"/"Heavy-K feat. Davido"
  AMBIGUOUS           Primary artists differ after full normalization.
                      Never auto-merged; written to review report only.

Both SAFE_ALIAS and SAME_PRIMARY_COLLAB are applied in --apply mode.
AMBIGUOUS groups are always skipped.

Normalization for comparison (_normalize_primary_for_compare):
  NFC unicode → lowercase → strip periods/apostrophes/quotes → hyphen/underscore→space
  → collapse spaces. This is applied AFTER extract_primary_artist() so collab suffixes
  do not affect the comparison.

Canonical name selection (_pick_canonical):
  Most files wins. Tiebreakers: penalise all-uppercase (HOSH < H.O.S.H),
  penalise all-lowercase (culoe < Culoe De Song), then alphabetical.
  Collab folder names are never chosen as canonical.

Report: logs/reports/artist_merge/artist_merge_{dry_run|applied}.json
  Sections: safe_alias_merges, same_primary_collab_merges, ambiguous_merges

Tests: tests/test_artist_merge.py — 77 tests

CONVERT AUDIO
----------------------------------------------------------------------
convert-audio subcommand: .m4a → .aiff, parallel ffmpeg workers.

Workflow:
  scan --src for .m4a → ffprobe validate → ffmpeg pcm_s16be → verify → move to --archive

Key behaviors:
  - Output codec: pcm_s16be (16-bit big-endian PCM AIFF)
  - Metadata copied via -map_metadata 0
  - Relative folder structure preserved in both --dst and --archive
  - Duration verification: abs(dst_dur - src_dur) <= --verify-tolerance-sec (default 1.0s)
  - On failure: original left untouched, broken output deleted
  - Archive move only happens after verification passes
  - Parallelism: ThreadPoolExecutor, N workers (default 4)
  - Progress: tqdm if installed, graceful fallback

Config:
  FFMPEG_BIN  = os.environ.get("FFMPEG_BIN",  "ffmpeg")   # new in v1.2.0
  FFPROBE_BIN = os.environ.get("FFPROBE_BIN", "ffprobe")  # existing
